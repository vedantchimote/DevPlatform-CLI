## Security Principles

<AccordionGroup>
  <Accordion title="Least Privilege Access">
    Grant minimum permissions required for each user, service, and application. Use IAM policies (AWS) or RBAC roles (Azure) to enforce granular access control.
    
    - Developers: Read-only access to production
    - Applications: Specific resource permissions only
    - Admins: Time-limited elevated access
  </Accordion>

  <Accordion title="Defense in Depth">
    Multiple layers of security controls to protect against various attack vectors:
    
    - Network layer: Security groups, NACLs, private subnets
    - Application layer: Authentication, authorization, input validation
    - Data layer: Encryption, access controls, audit logging
  </Accordion>

  <Accordion title="Zero Trust Network">
    Never trust, always verify. All network traffic is authenticated and authorized:
    
    - No implicit trust based on network location
    - Mutual TLS between services
    - Service-to-service authentication
  </Accordion>

  <Accordion title="Encryption Everywhere">
    Encrypt data at rest and in transit using industry-standard algorithms:
    
    - TLS 1.2+ for all network communication
    - AES-256 for data at rest
    - Customer-managed encryption keys
  </Accordion>
</AccordionGroup>

## Quick Security Checklist

<Steps>
  <Step title="Configure Cloud Authentication">
    Set up AWS CLI or Azure CLI with appropriate credentials
    
    <Tabs>
      <Tab title="AWS">
        ```bash
        aws configure
        # Enter Access Key ID and Secret Access Key
        ```
      </Tab>
      <Tab title="Azure">
        ```bash
        az login
        # Follow browser authentication flow
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Verify IAM Permissions">
    Ensure your user/role has required permissions for infrastructure provisioning
    
    <Tabs>
      <Tab title="AWS">
        ```bash
        aws sts get-caller-identity
        # Verify account and role
        ```
      </Tab>
      <Tab title="Azure">
        ```bash
        az account show
        # Verify subscription and tenant
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Enable Encryption">
    Verify encryption is enabled for all data stores
    
    - Database encryption at rest
    - Storage encryption (S3/Blob Storage)
    - Secrets encryption (Secrets Manager/Key Vault)
  </Step>

  <Step title="Configure Network Security">
    Set up security groups and network isolation
    
    - Private subnets for databases
    - Security group rules for least privilege
    - Network ACLs for additional protection
  </Step>

  <Step title="Enable Audit Logging">
    Turn on comprehensive logging for compliance
    
    <Tabs>
      <Tab title="AWS">
        - CloudTrail for API calls
        - VPC Flow Logs for network traffic
        - Kubernetes audit logs
      </Tab>
      <Tab title="Azure">
        - Activity Log for API calls
        - NSG Flow Logs for network traffic
        - Kubernetes audit logs
      </Tab>
    </Tabs>
  </Step>
</Steps>

## Security by Default

DevPlatform CLI implements security best practices automatically:

| Feature | AWS Implementation | Azure Implementation |
|---------|-------------------|---------------------|
| **Network Isolation** | Private subnets for databases and applications | Private subnets with service endpoints |
| **Encryption at Rest** | KMS encryption for RDS, EBS, S3 | Platform-managed encryption for all resources |
| **Encryption in Transit** | TLS 1.2+ for all connections | TLS 1.2+ for all connections |
| **Secret Management** | AWS Secrets Manager with KMS | Azure Key Vault with managed keys |
| **Access Control** | IAM roles with least privilege | Azure RBAC with least privilege |
| **Audit Logging** | CloudTrail + VPC Flow Logs | Activity Log + NSG Flow Logs |

<Note>
All security features are enabled by default. No additional configuration required for basic security posture.
</Note>

## Incident Response

<Steps>
  <Step title="Detection">
    Security event detected through monitoring or alerts
  </Step>
  <Step title="Isolation">
    Immediately revoke compromised credentials and isolate affected resources
  </Step>
  <Step title="Investigation">
    Analyze logs to determine scope and root cause
  </Step>
  <Step title="Remediation">
    Apply fixes, rotate credentials, patch vulnerabilities
  </Step>
  <Step title="Recovery">
    Restore services and verify security posture
  </Step>
  <Step title="Post-Mortem">
    Document incident and update security policies
  </Step>
</Steps>
