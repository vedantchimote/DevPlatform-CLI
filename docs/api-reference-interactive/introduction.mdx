## Overview

DevPlatform CLI provides a simple, intuitive command-line interface for provisioning and managing infrastructure environments on AWS and Azure.

## Command Structure

All commands follow this general structure:

```bash
devplatform <command> [flags]
```

## Common Patterns

### Basic Usage

```bash
# Create an environment
devplatform create --app myapp --env dev --provider aws

# Check status
devplatform status --app myapp --env dev

# Destroy environment
devplatform destroy --app myapp --env dev --confirm
```

### With Configuration File

```bash
# Use custom config file
devplatform create \
  --app myapp \
  --env dev \
  --config-file ./custom-config.yaml
```

### Verbose Output

```bash
# Enable verbose logging
devplatform create \
  --app myapp \
  --env dev \
  --verbose

# Enable debug logging
devplatform create \
  --app myapp \
  --env dev \
  --debug
```

### CI/CD Usage

```bash
# Disable colors for CI/CD
devplatform create \
  --app myapp \
  --env dev \
  --no-color \
  --confirm
```

## Exit Codes

All commands use consistent exit codes:

| Code Range | Category | Description |
|------------|----------|-------------|
| 0 | Success | Command completed successfully |
| 1 | General Error | Unspecified error occurred |
| 1000-1099 | Authentication | Credential or permission errors |
| 1100-1199 | Validation | Input validation errors |
| 1200-1299 | Terraform | Infrastructure provisioning errors |
| 1300-1399 | Helm | Application deployment errors |
| 1400-1499 | Network | Network connectivity errors |
| 1500-1599 | Configuration | Configuration file errors |
| 2000-2099 | Azure | Azure-specific errors |

### Example Exit Code Handling

```bash
#!/bin/bash

devplatform create --app myapp --env dev
EXIT_CODE=$?

if [ $EXIT_CODE -eq 0 ]; then
  echo "Success!"
elif [ $EXIT_CODE -ge 1000 ] && [ $EXIT_CODE -le 1099 ]; then
  echo "Authentication error. Check your credentials."
elif [ $EXIT_CODE -ge 1200 ] && [ $EXIT_CODE -le 1299 ]; then
  echo "Terraform error. Check the logs."
else
  echo "Unknown error: $EXIT_CODE"
fi
```

## Environment Variables

DevPlatform CLI respects these environment variables:

<AccordionGroup>
  <Accordion title="DEVPLATFORM_CONFIG">
    Path to configuration file (overrides default)
    
    ```bash
    export DEVPLATFORM_CONFIG=/path/to/config.yaml
    ```
  </Accordion>

  <Accordion title="DEVPLATFORM_LOG_LEVEL">
    Set log level: debug, info, warn, error
    
    ```bash
    export DEVPLATFORM_LOG_LEVEL=debug
    ```
  </Accordion>

  <Accordion title="DEVPLATFORM_NO_COLOR">
    Disable colored output
    
    ```bash
    export DEVPLATFORM_NO_COLOR=1
    ```
  </Accordion>

  <Accordion title="AWS_* Variables">
    Standard AWS environment variables are respected:
    - AWS_ACCESS_KEY_ID
    - AWS_SECRET_ACCESS_KEY
    - AWS_SESSION_TOKEN
    - AWS_REGION
    - AWS_PROFILE
  </Accordion>

  <Accordion title="AZURE_* Variables">
    Standard Azure environment variables are respected:
    - AZURE_SUBSCRIPTION_ID
    - AZURE_TENANT_ID
    - AZURE_CLIENT_ID
    - AZURE_CLIENT_SECRET
  </Accordion>
</AccordionGroup>

## Configuration Precedence

Configuration values are resolved in this order (highest to lowest priority):

1. **Command-line flags** (highest priority)
2. **Environment variables**
3. **Configuration file** (specified with --config-file)
4. **Default configuration file** (~/.devplatform/config.yaml)
5. **Built-in defaults** (lowest priority)

### Example

```bash
# Config file sets region to us-east-1
# Environment variable sets region to us-west-2
export AWS_REGION=us-west-2

# Command-line flag sets region to eu-west-1 (this wins)
devplatform create --app myapp --env dev --region eu-west-1
```

## Examples

### Complete Workflow

```bash
# 1. Create environment
devplatform create --app payment --env dev --provider aws

# 2. Check status
devplatform status --app payment --env dev

# 3. Update application (re-run create)
devplatform create --app payment --env dev --values-file updated-values.yaml

# 4. Destroy when done
devplatform destroy --app payment --env dev --confirm
```

### Multi-Environment Management

```bash
# Create dev environment
devplatform create --app payment --env dev

# Create staging environment
devplatform create --app payment --env staging

# Create production environment
devplatform create --app payment --env prod

# Check all environments
devplatform status --app payment --env dev
devplatform status --app payment --env staging
devplatform status --app payment --env prod
```

### CI/CD Integration

```bash
#!/bin/bash
set -e

# Deploy to staging on merge to main
if [ "$BRANCH" = "main" ]; then
  devplatform create \
    --app $APP_NAME \
    --env staging \
    --provider aws \
    --no-color \
    --timeout 10
fi

# Deploy to production on tag
if [ -n "$TAG" ]; then
  devplatform create \
    --app $APP_NAME \
    --env prod \
    --provider aws \
    --no-color \
    --timeout 15
fi
```

## Best Practices

<AccordionGroup>
  <Accordion title="Use Configuration Files">
    Store common settings in configuration files instead of passing flags every time:

    ```yaml
    # ~/.devplatform/config.yaml
    global:
      cloud_provider: aws
      timeout: 30

    aws:
      region: us-east-1
      profile: default
    ```
  </Accordion>

  <Accordion title="Version Control Your Config">
    Keep configuration files in version control:

    ```bash
    git add .devplatform/config.yaml
    git commit -m "Add DevPlatform configuration"
    ```
  </Accordion>

  <Accordion title="Use Dry Run">
    Always test with --dry-run first:

    ```bash
    devplatform create --app myapp --env prod --dry-run
    ```
  </Accordion>

  <Accordion title="Enable Verbose Logging for Debugging">
    Use --verbose or --debug when troubleshooting:

    ```bash
    devplatform create --app myapp --env dev --debug
    ```
  </Accordion>

  <Accordion title="Automate with Scripts">
    Create wrapper scripts for common operations:

    ```bash
    #!/bin/bash
    # deploy.sh
    devplatform create \
      --app $1 \
      --env $2 \
      --provider ${CLOUD_PROVIDER:-aws} \
      --verbose
    ```
  </Accordion>
</AccordionGroup>
