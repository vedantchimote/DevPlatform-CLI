## Overview

The `version` command displays version information for DevPlatform CLI and optionally checks all required dependencies.

## Syntax

```bash
devplatform version [flags]
```

## Optional Flags

<ParamField path="short" type="boolean" default="false">
  Display only the version number
</ParamField>

<ParamField path="check-deps" type="boolean" default="false">
  Check all dependency versions and requirements
</ParamField>

<ParamField path="output" type="string" default="text">
  Output format
  
  **Options:** `text`, `json`, `yaml`
</ParamField>

## Examples

### Basic Version

```bash
devplatform version
```

Output:

```
DevPlatform CLI v1.0.0

Build Information:
  Git Commit: a1b2c3d
  Build Date: 2024-01-15T10:00:00Z
  Go Version: go1.21.5
  Platform: darwin/arm64

For more information: https://docs.devplatform.io
```

### Short Version

```bash
devplatform version --short
```

Output:

```
v1.0.0
```

Useful for scripting:

```bash
VERSION=$(devplatform version --short)
echo "Running DevPlatform CLI $VERSION"
```

### Check Dependencies

```bash
devplatform version --check-deps
```

Output:

```
DevPlatform CLI: v1.0.0

Dependency Versions:
  ✓ Terraform: v1.6.0 (minimum: v1.5.0)
  ✓ Helm: v3.13.0 (minimum: v3.0.0)
  ✓ kubectl: v1.28.0 (minimum: v1.27.0)
  ✓ AWS CLI: v2.13.25 (minimum: v2.0.0)
  ✓ Azure CLI: v2.53.0 (minimum: v2.0.0)

✅ All dependencies satisfied!

Cloud Provider Status:
  ✓ AWS: Credentials configured (us-east-1)
  ✓ Azure: Authenticated (subscription: 12345678-1234-1234-1234-123456789012)
```

└───── Major: Breaking changes

## Use Cases

### CI/CD Version Check

```bash
#!/bin/bash
# check-version.sh

REQUIRED_VERSION="v1.0.0"
INSTALLED_VERSION=$(devplatform version --short)

if [ "$INSTALLED_VERSION" != "$REQUIRED_VERSION" ]; then
  echo "Error: DevPlatform CLI version mismatch"
  echo "Required: $REQUIRED_VERSION"
  echo "Installed: $INSTALLED_VERSION"
  exit 1
fi

echo "Version check passed: $INSTALLED_VERSION"
```

### Dependency Validation

```bash
#!/bin/bash
# validate-dependencies.sh

# Check all dependencies
devplatform version --check-deps --output json > deps.json

# Parse results
TERRAFORM_OK=$(jq -r '.dependencies.terraform.satisfied' deps.json)
HELM_OK=$(jq -r '.dependencies.helm.satisfied' deps.json)
KUBECTL_OK=$(jq -r '.dependencies.kubectl.satisfied' deps.json)

if [ "$TERRAFORM_OK" != "true" ] || [ "$HELM_OK" != "true" ] || [ "$KUBECTL_OK" != "true" ]; then
  echo "Error: Missing or outdated dependencies"
  cat deps.json
  exit 1
fi

echo "All dependencies satisfied"
```

### Version Reporting

```bash
#!/bin/bash
# report-versions.sh

# Generate version report
cat << EOF
DevPlatform Environment Report
==============================

CLI Version: $(devplatform version --short)

Dependencies:
$(devplatform version --check-deps | grep "✓")

Cloud Providers:
$(devplatform version --check-deps | grep "Cloud Provider Status" -A 10)

Generated: $(date)
EOF
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Dependency Not Found">
    If a dependency is not found:

    ```
    ✗ Terraform: not found (minimum: v1.5.0)
    ```

    **Solution:**
    1. Install the missing dependency
    2. Ensure it's in your PATH
    3. Verify installation: `which terraform`
  </Accordion>

  <Accordion title="Version Too Old">
    If a dependency version is too old:

    ```
    ✗ Helm: v2.17.0 (minimum: v3.0.0)
    ```

    **Solution:**
    1. Upgrade the dependency
    2. Check for conflicts with other tools
    3. Verify upgrade: `helm version`
  </Accordion>

  <Accordion title="Cloud Provider Not Configured">
    If cloud credentials are not configured:

    ```
    ✗ AWS: Credentials not configured
    ```

    **Solution:**
    ```bash
    # Configure AWS
    aws configure
    
    # Or Azure
    az login
    ```
  </Accordion>
</AccordionGroup>

## Exit Codes

| Code | Meaning |
|------|---------|
| 0 | Success (all dependencies satisfied if --check-deps) |
| 1 | Missing or outdated dependencies (with --check-deps) |
